!! Portuguese version below

This build alone is a pain in the ass,
There are 2 releases of this build with different names, but one of them is actually 2. R2 has a version compiled right after R1
with the goal of fixing a specific bug related to ubiquity, which prevented the system from installing, consequently
it wasn't possible to see how Plasma looked inside R1. In 2097 R2, a line of code from the LuminOS LiveCD was corrected
but this still didn't fix the ubiquity problem. This all happened at the beginning of December 2025, 2 days after the compilation of 2093
as fulfillment of the promise that the first version of LuminOS would come in December, a promise that was made in the markdown explaining
the project's closure under the name "NexusOS" and the announcement of the new name "LuminOS".

After that, jumping to January, I decided to move the LuminOS base from Ubuntu to Kubuntu, thinking that this might solve the bug of the system starting in GNOME and not Plasma by default.

If you still haven't understood the problem with this build, it has been in development since December of last year, still without any stable compilation.
Besides being one of the builds that brings the most changes, because 2093 was just a test build, which brought KDE Plasma and the official Milestone "Eclipse" wallpaper.
While build 2097 brings the features that are actually the purpose of LuminOS, to use .NET as a compiler.

I created this whole text file to kind of "vent" about it, even though I have nothing to complain about,
because build 2093 was even harder to do, because I (the "jenius" that I am) decided to try compiling it through WSL, twice
since I was focusing on some other things, and couldn't leave Windows at the moment. But when I finally could leave it,
I switched to Kubuntu, where I managed to compile LuminOS 2093, and where I am compiling 2097.
But the other reason I created this text file was to somehow "save" the new folder organization of 2097,
merely so I wouldn't end up getting confused and creating 20 copies of the same folder.

Inside the /home folder, where the users are located, we have the luminos folder, which contains some important files, such as the one for creating a new user.
Inside it, we have the LuminOS folder, which is the default system container (since each user has their own container).
This folder isn't used very often, but in beta versions, to unlock everything LuminOS has to offer, you have to run the new user command.

This new user is located inside the /home/luminos folder.

If it wasn't clear, the general system container is located in /home/luminos, and inside it, just like in ~/container, we have the LuminOS folder.
